# Demo Pitch — 头号Builder全球挑战赛 · 北京站

## 项目：DocTransAgent — 海外文档智能翻译与知识库构建 Agent

---

## 演示流程（5 分钟）

### 0. 开场 Hook（30 秒）

> 中国企业出海每年产生数十万份中文文档——技术手册、品牌指南、合规文件、培训材料。翻译成本高、周期长、术语不一致、知识无法沉淀。我们的解决方案：基于 GMI Cloud 多模型推理引擎的 DocTransAgent，一键翻译 + 知识库 + 智能问答。

**屏幕展示**：仪表盘页面，显示 GMI Cloud 四模型架构（Gemini/DeepSeek/GLM/Qwen）

---

### 1. 文档上传与智能解析（60 秒）

**操作**：
1. 拖拽上传 3 份中文文档：产品技术手册、品牌战略手册、合规与数据保护政策
2. 系统自动识别文件格式（MD/PDF/DOCX）
3. 展示解析结果：字数、段落结构、标题层级

**话术**：
> 支持 PDF、DOCX、Markdown、TXT 多格式。文档上传后自动解析，保留标题层级和段落结构，为后续精准翻译打下基础。

**评委动效点**：
- 3 份不同业务类型文档同时出现（技术/市场/法律）
- 显示解析出的章节结构

---

### 2. 术语表驱动的智能翻译（90 秒）

**操作**：
1. 点击"翻译"，后台调用 GMI Cloud Gemini 2.5 Flash
2. 展示实时翻译进度条（并行翻译多分块）
3. 切换到术语表页面，展示预设的企业术语
4. 对比开关术语表的翻译效果：
   - 无术语表："智能屏 X1 Pro" → "Smart Screen X1 Pro"
   - 有术语表："智能屏 X1 Pro" → "SmartScreen X1 Pro"（品牌一致）

**话术**：
> 这是我们的核心差异化之一：术语一致性引擎。企业只需维护一份术语表——品牌名、产品名、技术词汇——系统自动注入翻译提示词，确保所有文档翻译统一。你可以看到 Gemini 2.5 Flash 通过 GMI Cloud 在 5 秒内完成了这个段落的翻译。

**评委动效点**：
- 翻译进度条实时滚动（展示并行能力）
- 术语表开关 A/B 对比
- ModelBadge 显示 "Gemini 2.5 Flash · 234ms · 1204 tok"

---

### 3. 跨语言知识库搜索（90 秒）

**操作**：
1. 翻译完成后一键索引到向量知识库
2. 搜索"产品合规要求"（中文）→ 结果展示英文合规文档的内容
3. 搜索"What certifications for EU market"（英文）→ 结果展示中文技术手册

**话术**：
> 这是展示 GMI Cloud 多模型能力的关键环节。我们使用 Qwen3-Embedding-8B 多语言嵌入模型，通过 GMI Cloud 生成向量。这意味着——用中文搜索能找到英文文档，反之亦然。企业出海最痛苦的"不知道英文文档写在哪"的问题，我们解决了。

**评委动效点**：
- 关键词高亮在搜索结果中
- 显示匹配百分比
- 中→英 / 英→中 双向搜索

---

### 4. RAG 智能问答（90 秒）

**操作**：
1. 打开 Q&A 页面
2. 提问："Our product needs to enter the EU market — what certifications are required?"（英文）
3. DeepSeek V3 流式输出回答，带 `[Source 1]` `[Source 2]` 引用标记
4. 追问中文："这些认证大概需要多久办好？"
5. 系统理解上下文，跨语言回答

**话术**：
> 问答引擎使用 DeepSeek V3 通过 GMI Cloud。我们从知识库中检索最相关的 5 个文档片段，构建 RAG 提示词。注意回答中的 Source 标记——点击可以追溯到原始文档段落。这就是 RAG 的魔力：每个答案都有据可查。

**评委动效点**：
- SSE 流式输出
- Source 引用高亮
- 跨语言追问

---

### 5. ROI & 总结（60 秒）

**操作**：
1. 切换到仪表盘
2. 展示统计数据：3 份文档、约 8000 字、翻译成本 $0.18、人工翻译对比节省 $640

**话术**：
> 两天 hackathon，一个能用的产品。感谢 GMI Cloud 提供的多模型推理引擎——让我们在一套 API 下调用四个最合适的模型，从翻译到嵌入到问答，无切换成本。对于出海企业，这意味着每年节省数十万美元的文档管理与翻译成本，同时确保品牌信息和合规内容的准确统一。
